## 🤝 贡献指南

我们欢迎社区贡献！以下是您可以提供帮助的方式：

### 🚀 快速开始

1. **Fork** 仓库
2. **克隆**您的 fork 到本地
3. **创建**功能分支：`git checkout -b feature/amazing-feature`
4. **进行**您的修改
5. **充分测试**
6. **提交**清晰的提交信息：`git commit -m '添加 amazing 功能'`
7. **推送**到您的分支：`git push origin feature/amazing-feature`
8. **创建** Pull Request

### 📋 贡献指南

#### 🐛 Bug 报告

- 使用 [Bug 报告模板](https://github.com/TracingOrigins/obsidian-music-player-plugin/issues/new?template=bug_report.md)
- 包含复现步骤
- 描述预期行为与实际行为
- 提供系统信息（操作系统、Obsidian 版本、插件版本）

#### 💡 功能请求

- 使用 [功能请求模板](https://github.com/TracingOrigins/obsidian-music-player-plugin/issues/new?template=feature_request.md)
- 解释您要解决的问题
- 描述您提出的解决方案
- 考虑实现复杂度

#### 🔧 代码贡献

- 遵循现有的代码风格和约定
- 为新功能添加测试
- 根据需要更新文档
- 确保所有测试通过后再提交

#### 📚 文档贡献

- 改进 README 文件
- 添加代码注释
- 创建教程或指南
- 翻译成其他语言

### 🛠️ 开发环境设置

```bash
# 克隆仓库
git clone https://github.com/TracingOrigins/obsidian-music-player-plugin.git
cd obsidian-music-player-plugin

# 安装依赖
npm install

# 启动开发模式
npm run dev

# 构建生产版本
npm run build

# 运行测试（如果可用）
npm test
```

### 📝 提交信息规范

使用约定式提交格式：

- `feat:` 新功能
- `fix:` Bug 修复
- `docs:` 文档更改
- `style:` 格式更改
- `refactor:` 代码重构
- `test:` 添加测试
- `chore:` 维护任务

### 🎯 需要帮助的领域

- 🌍 **翻译**: 帮助翻译成更多语言
- 🧪 **测试**: 添加单元测试和集成测试
- 📖 **文档**: 改进指南和示例
- 🎨 **UI/UX**: 增强用户界面
- 🔧 **性能**: 优化代码并减少包大小
